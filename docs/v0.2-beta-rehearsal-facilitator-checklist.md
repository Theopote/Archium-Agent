## 时间轴（90–120 min）

| 时段 | 用户做什么 | 你记录什么 |
|------|------------|------------|
| 0–15 min | 新建项目 → 上传资料 → 看片段/事实账本 | 导入是否卡顿、路径/编码问题 → `beta-issue-triage.csv` |
| 15–35 min | 填 Brief → 审 Brief → Storyline → SlideSpec → 点生成/导出 | 是否理解审核含义；导出选项勾选：**JSON + 原生 PPTX**（或团队统一 Marp） |
| 35–75 min | 在 **工作台 + PowerPoint** 改稿 | **每改完一页**在 `beta-edit-cost-sheet.csv` 记：类别 + 分钟 + 是否 blocking |
| 75–85 min | 最终导出，用 PowerPoint 打开终稿 | `export` 类问题；能否拿去汇报（是/否） |
| 85–100 min | 口头复盘 | 每条问题写入 triage 三档 |

## 修改类别（记 CSV 用）

`text` · `layout` · `image` · `fact_citation` · `structure` · `export` · `other`

## 问题三档（记 triage 用）

| 档位 | 一句话 |
|------|--------|
| **beta_blocker** | 不修就不能叫 Beta |
| **post_beta_improvement** | 难受但能交付 |
| **future_idea** | 以后再说 |

## 会中禁忌

- 不要替用户点「生成」或改 Brief/Storyline 正文
- 不要现场改代码（问题先记 triage，会后再修）
- 不要跳过 PowerPoint 打开终稿

## 会后 15 分钟（工程师）

```powershell
python scripts/summarize_beta_rehearsal.py docs/rehearsal/sessions/<session_id>/ --output docs/rehearsal/sessions/<session_id>/summary.json
```

- [ ] `summary.json` 中 `participants_non_dev >= 1`
- [ ] `total_edit_minutes > 0`
- [ ] 所有 **critical/high** 已在 triage 表
- [ ] 开放 `beta_blocker` 数记入 [v0.2-beta-release-decision.md](v0.2-beta-release-decision.md)
- [ ] 若 `beta_ready_by_user_data: true` 且 CI 绿 → 提议打 `v0.2-beta.1` tag

## 紧急联系

| 现象 | 先做什么 |
|------|----------|
| 无法启动 | 检查 `.env`、无 Key 是否仅 LLM 步骤失败 |
| PPTX 打不开 | 记 `export` + beta_blocker；保留 `data/outputs/` 文件 |
| 中文路径失败 | 记 triage；对照 [cross-platform-validation.md](cross-platform-validation.md) |
