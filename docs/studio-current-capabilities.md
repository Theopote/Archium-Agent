# Studio 当前能力补充

> 当前代码事实快照：2026-07-23。本文补充 [汇报工作室用户指南](studio-user-guide.md) 尚未覆盖的新能力；完成主指南合并后可删除。

## 检查器

当前 Studio 右侧检查器共有七个 Tab：**属性、布局、内容、AI、评论、风格、检查**。

- **属性**：单元素属性与多选批量几何操作。
- **布局**：候选版式、当前页重排与方案切换。
- **内容**：内容适配建议与固定画布容量预算。
- **AI**：把元素、多选、区域或整页评论编译为修改提案。
- **评论**：统一 Inbox、筛选、定位、rebase 与解决。
- **风格**：整套主题、字体、颜色与图片处理策略。
- **检查**：确定性/语义问题和人工视觉复核。

## 画布选择与批量操作

Canvas Editor 支持点击单选、`Shift` 多选、拖拽框选、双击改字或换图。多选后可执行对齐、分布、等宽和等高。画布操作修改 RenderScene，并进入正式修订历史，而不是直接覆写导出的 PPTX。

## 评论、提案与版本

评论作用域包括：

| 作用域 | 行为 |
|---|---|
| `node` | 只允许修改硬绑定节点，默认值 |
| `node_and_references` | 节点及显式关联节点 |
| `selection` | 当前多选节点集合 |
| `region` | 由选中节点包围盒定义的区域 |
| `slide` | 当前整页 |

评论保存创建时的 Scene revision、hash 和节点快照。当前正式 Scene 已变化或绑定节点消失时，评论进入 `needs_rebase`；重新绑定当前版本后才能继续生成或接受提案。评论 Inbox 提供状态计数、页码/作者/状态筛选、排序和画布定位。

## 固定画布容量门禁

预布局阶段计算 `SlideCapacityBudget`：

| 状态 | 含义与动作 |
|---|---|
| `fits` | 内容适配固定画布，可继续 |
| `tight` | 可生成候选，但必须进行 QA |
| `overloaded` | 禁止继续缩字；缩短内容或拆页 |
| `impossible` | 当前载荷无法放入画布；必须换版式或重构页面 |

预算包含真实/回退字体度量审计，并为建筑图纸单独保留最低可读面积、图注、图例/指北针/比例尺和标注密度空间。

## 图片衍生处理

图片处理通过 `ImageTreatmentSpec -> ImageDerivative` 创建不可变衍生文件，不覆盖原始资产。当前 Pillow 执行路径支持安全标准化、演示统一和文档扫描处理。项目图纸与项目证据照片只允许 `none` 或 `safe_normalize`，避免表达性处理改变证据含义。

## 字体与渲染安全

Scene 可声明字体资产与回退链。语义 QA 会提示未绑定字体可能造成的渲染端回退和重排风险。Renderer 负责执行已确定的 LayoutPlan/RenderScene；不应在 PptxGenJS 中重新选择版式或偷偷处理原图。
