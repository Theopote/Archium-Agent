# Archium Beta 试用 — 参与者说明（非开发人员）

感谢参与本次试用。你的任务是**像平时做汇报一样**使用 Archium，我们会记录你改稿花了多少时间，用来判断产品是否达到 Beta 标准。

## 你需要准备

- Windows 电脑，已安装 **Microsoft PowerPoint**
- 一份**已脱敏**的项目资料（PDF、Word、PPT、图片均可）
- 约 **90–120 分钟** uninterrupted 时间

## 你会做什么

1. **导入资料** — 在网页里新建项目，上传你的文件。
2. **填写汇报要求** — 按提示说明汇报对象、目的、页数等（Brief）。
3. **审核 AI 草案** — 确认或修改 Brief、叙事结构（Storyline）、每页内容（SlideSpec）。
4. **生成并导出** — 得到汇报文件（含可在 PowerPoint 打开的 PPTX）。
5. **改稿** — 在网页和 PowerPoint 里修改，直到你觉得「可以拿去汇报」或「实在改不动了」。

## 我们需要你配合记录的

工作人员会请你**每改完一页**简单说一下：

- 改了什么（文字 / 版式 / 图片 / 数据引用等）
- 大概花了多少分钟
- 有没有「不改就没法交差」的问题

你**不需要**懂技术术语，用日常语言描述即可。

## 请注意

- 生成和审核步骤请**你自己操作**；工作人员只旁观和记录，不会代你点按钮。
- 遇到卡住或看不懂的地方，**先说出来**，我们会记下来，不会当场改软件。
- 资料请勿包含未脱敏的业主名称、合同金额等敏感信息。

## 成功标准（对你而言）

走完：**上传资料 → 生成汇报 → 自己改稿 → 导出并在 PowerPoint 打开**。  
不要求一次完美，但要求流程能走通。

如有疑问，请联系本次 session 的 Facilitator。
