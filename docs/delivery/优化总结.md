### ✅ 2. 修复通用异常捕获
**状态**: 完成  
**修改的文件**:

   - 将 `except Exception:` 改为 `except (ImportError, RuntimeError):`

**改进**:
- 异常类型更加具体，便于调试
- 添加了注释说明捕获的原因

### ✅ 4. 添加 pre-commit hooks
**状态**: 完成  
**创建的文件**:
1. `.pre-commit-config.yaml`
   - 基础文件检查（YAML/JSON/TOML 验证）
   - 防止大文件提交（>1MB）
   - 尾随空格和换行符修复
   - 合并冲突检测

2. `.secrets.baseline` (7.5KB)
   - detect-secrets 基线文件
   - 扫描了整个代码库的现有"疑似密钥"

3. Ruff 代码质量检查
   - 自动修复代码风格问题
   - 自动格式化

**安装的工具**:
- pre-commit 4.6.0
- detect-secrets 1.5.0

**使用方法**:
```bash
# 安装 hooks（首次）
pre-commit install

# 手动运行所有检查
pre-commit run --all-files

# 正常提交会自动运行
git commit -m "message"
```

### ✅ 5. 添加资源限制配置
**状态**: 完成  
**修改的文件**: `archium/config/settings.py`

**新增配置项**:

1. **LLM 并发控制**:
```python
llm_max_concurrent_requests: int = Field(
    default=5,
    ge=1,
    le=20,
    description="Maximum concurrent LLM API requests..."
)
```

2. **ChromaDB 文档上限**:
```python
chroma_max_documents: int = Field(
    default=10000,
    ge=100,
    description="Maximum number of documents in Chroma vector store..."
)
```

3. **Checkpoint 清理策略**:
```python
workflow_checkpoint_retention_days: int = Field(
    default=7,
    ge=1,
    le=90,
    description="Auto-delete workflow checkpoints older than this many days..."
)
```

**效果**:
- 防止 LLM API 过度调用和速率限制
- 限制向量数据库内存占用
- 自动清理旧的工作流 checkpoint，节省存储空间

## 团队协作建议

### 给开发者
1. **首次克隆后**:
   ```bash
   git clone <repo>
   python -m venv .venv
   source .venv/bin/activate  # Windows: .venv\Scripts\activate
   pip install -e ".[full,dev]"
   pre-commit install
   cp .env.example .env
   # 编辑 .env 填入你的 API keys
   ```

2. **提交代码前**:
   ```bash
   pre-commit run --all-files  # 检查所有文件
   pytest                       # 运行测试
   ```

3. **遇到 pre-commit 失败**:
   - Ruff 会自动修复大部分问题
   - detect-secrets 误报可以更新 .secrets.baseline
   - 不要使用 `--no-verify` 跳过检查

### 给项目维护者
1. 定期更新 pre-commit hooks:
   ```bash
   pre-commit autoupdate
   ```

2. 审查 .secrets.baseline 变更:
   - 新增的"密钥"可能是真实泄露
   - 需要人工确认

3. 监控 pyproject.toml 依赖变化:
   - 大版本升级需要测试
   - 新增依赖需要评审
