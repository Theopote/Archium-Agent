# Hospital renovation report

## Intent

医院改造汇报必须让院方 / 建设方看懂：**不停业如何成立、流线与感染风险如何控、分期如何落地**。对齐 golden 语境 `case_a_hospital`。

## Narrative spine

1. **现状问题**（证据板：拥堵、交叉、老化、后勤瓶颈）
2. **约束**（不停业、消防、感染控制、规范、投资边界）— 来自事实账本
3. **策略** — 每条策略必须回应前文问题
4. **分期 / 过渡运营** — 临时流线、替代用房、施工界面
5. **总图与关键科室平面** — 指北针、比例尺、contain
6. **指标与请求决策** — 有来源；明确需批准事项

## Domain checklist

```
- [ ] 不停业 / 分区实施说清楚
- [ ] 医患 / 洁污 / 急救流线无未解释交叉
- [ ] 感染控制或隔离逻辑有交代（若相关）
- [ ] 策略 ↔ 问题编号对应
- [ ] 总平面含指北针与比例尺
- [ ] 图纸 contain；参考案例未冒充本院证据
- [ ] 无连续大量同构卡片页
```
