# Campus renovation report

## Intent

校园改造汇报聚焦：**功能置换是否成立、步行与开放共享如何改善、分期如何少扰教学**。对齐 golden 语境 `case_b_campus`。

## Narrative spine

1. **现状问题**（老化、功能错配、步行断裂、开放不足）— 证据可追溯
2. **约束**（教学不停、文物/风貌、消防、投资）— 事实账本优先
3. **策略** — 必须回应前文问题（置换、织补、共享、景观/入口）
4. **总图与关键建筑平面** — 指北针、比例尺、contain
5. **分期实施** — 对教学周历 / 动线干扰的说明
6. **指标与决策请求** — 有来源

## Domain checklist

```
- [ ] 功能置换逻辑清楚（从→到）
- [ ] 步行体系 / 开放共享有图或证据支撑
- [ ] 策略 ↔ 问题对应
- [ ] 分期对教学运营的影响已说明
- [ ] 总平面含指北针与比例尺
- [ ] 图纸 contain；参考案例已标注
- [ ] 无连续大量同构卡片页
```
