# Executive summary writing

## Intent

开篇 / 结论 / 执行摘要：让决策者 **30 秒抓住主张与依据**，不是目录或口号墙。

## Structure

1. **一句核心主张**（中心结论）— 可辩论、可验证，非「坚持以人为本」类空话
2. **3–5 条支撑** — 每条对应项目事实或已确认证据；禁止无来源指标
3. **下一步 / 请求决策**（若适用）— 明确要什么决议（原则通过、分期预算、流线方案等）

## Hard constraints（继承总则）

- 一页一个中心结论
- 项目事实优先；未知标明未知
- 指标必须有来源
- 参考案例不得写成项目成果

## Anti-patterns

- 堆砌未证实数字（面积、造价、床位）
- 把整本大纲塞进一页
- 连续摘要页重复同一套卡片结构

## Checklist

```
- [ ] 核心主张一句话可复述
- [ ] 支撑点 ≤ 5，均有依据
- [ ] 无来源指标已删除或降级
- [ ] 请求决策清晰（如需要）
```
