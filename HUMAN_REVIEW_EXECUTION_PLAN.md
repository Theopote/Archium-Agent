## 为什么需要人工评审？

### 规则验证 vs 视觉质量

**规则验证（已完成）**：
```
✅ 元素不重叠
✅ 文字不溢出
✅ 字号符合标准
✅ Hero 元素面积充足
✅ 留白比例合理
```

**人工评审（未完成）**：
```
? 信息层次是否清晰
? 视觉焦点是否突出
? 阅读顺序是否自然
? 图文关系是否协调
? 留白密度是否舒适
? 建筑表达是否专业
? 美学完成度如何
? 可编辑性是否友好
```

### 类比

**规则验证** = 检查房子是否"结构安全"
- 墙没有裂缝 ✓
- 门窗能开关 ✓
- 水电通畅 ✓

**人工评审** = 检查房子是否"宜居舒适"
- 采光好不好？
- 布局合理吗？
- 住起来舒服吗？
- 客户愿意买吗？

## 人工评审流程设计

### 评审维度（8 个）

基于 `HUMAN_REVIEW_WEIGHTS`：

| 维度 | 权重 | 评分标准（1-5） |
|------|------|----------------|
| **information_hierarchy** | 15% | 信息层次清晰度 |
| **visual_focus** | 15% | 视觉焦点突出度 |
| **reading_order** | 10% | 阅读顺序自然度 |
| **image_text_relationship** | 15% | 图文关系协调度 |
| **whitespace_density** | 10% | 留白密度舒适度 |
| **architectural_expression** | 15% | 建筑表达专业度 |
| **aesthetic_finish** | 10% | 美学完成度 |
| **editability** | 10% | 可编辑性友好度 |

**通过阈值**：加权平均 ≥ 3.5 分

### 评审流程

```
1. 准备评审环境
   ├─ 生成 30 页截图（PNG）
   ├─ 准备评审表格
   └─ 确定评审人员

2. 逐页评审
   ├─ 查看页面截图
   ├─ 打开实际 PPTX 测试可编辑性
   ├─ 填写 8 个维度评分（1-5）
   ├─ 记录主要问题和次要问题
   └─ 判断是否接受

3. 汇总结果
   ├─ 计算加权平均分
   ├─ 统计通过率
   └─ 提取常见问题

4. 更新数据
   ├─ 保存评审记录到 JSON
   ├─ 更新 Benchmark 统计
   └─ 生成评审报告
```

**任务 1.1**: 生成截图

```python
# scripts/generate_benchmark_screenshots.py

from archium.application.visual.screenshot_service import ScreenshotService
from archium.application.visual.benchmark_service import BenchmarkService

def generate_screenshots():
    """为 30 个 Benchmark 案例生成截图"""
    service = BenchmarkService(session)
    screenshot_service = ScreenshotService(session)
    
    for case in benchmark_cases:
        result = service.build_case(case)
        
        # 生成截图
        screenshot_path = f"benchmark_data/screenshots/{case.case_id}.png"
        screenshot_service.capture_slide(
            result.plan,
            result.design_system,
            output_path=screenshot_path
        )
        
        print(f"✓ {case.case_id}: {screenshot_path}")
```

### Phase 2: 评审（4-6 小时）

**评审人员**：
- 建议：2-3 人独立评审

1. 打开截图浏览器

- 打分和记录：2-3 分钟
- 测试编辑：2-3 分钟

**总时间估算**：
- 单人：30 页 × 8 分钟 = 4 小时
